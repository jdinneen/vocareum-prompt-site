# **AI Notebook**

*Cloud-based Jupyter-powered learning and lab environment.*

## **Overview**

AI Notebook is Vocareum’s flagship notebook product for browser-based technical learning and governed hands-on work. It supports Jupyter-powered instruction, budgeted compute, autograding, and access to AI and GPU resources inside a controlled environment.

# **Cloud Labs**

*Governed hands-on cloud access for real platform experience.*

## **Overview**

Cloud Labs provide direct access to real cloud environments while preserving governance, budget controls, account isolation, and safety nets. They are used when a real platform experience matters more than a simulated interface.

# **AI Gateway**

*Governed model access and policy control for institutional AI use.*

## **Overview**

AI Gateway gives institutions and platforms a governed layer for model access, usage policy, and budget controls. It is used when organizations need model access without giving up operational control.

# **Developer Workspaces**

*Browser-based workspaces across multiple IDE and desktop surfaces.*

## **Overview**

Developer Workspaces provide browser-based working environments such as VS Code, JupyterLab, RStudio, Linux Desktop, and Windows VDI. They give learners or teams a ready-to-use workspace without requiring local setup.

# **AI Compass**

*Course-grounded AI tutoring and learner support tied to learner workflow.*

## **Overview**

AI Compass provides course-grounded tutoring, guidance, and learner support inside the same environment where learners practice and are assessed. It is designed to improve support quality without turning into a generic chatbot experience.

# **Agentic AI Labs**

*Browser-based IDE plus AI agent plus real cloud target in one governed sandbox.*

## **Overview**

Agentic AI Labs combine a browser-based IDE, an integrated AI agent, and real cloud targets inside one governed environment. They are used when organizations need hands-on agentic development and experimentation with policy, budget, and infrastructure controls intact.

# **On-the-Fly Labs**

*AI-generated hands-on labs created rapidly for a specific learner, cohort, workshop, or event.*

## **Overview**

On-the-Fly Labs use AI-assisted authoring to generate complete, deployable lab experiences in minutes rather than the weeks required by traditional manual authoring. Instructors, partners, or program leads describe a learning objective, target audience, and tooling context; the platform produces a structured lab with instructions, starter code, validation criteria, and environment configuration ready to launch. On-the-Fly Labs run on the same governed infrastructure as all other Vocareum lab types and environments are isolated, budgeted, and policy-controlled.

## **Core capabilities**

- AI-assisted lab generation from a learning objective, audience description, and tooling context
- Structured output including instructions, starter code or templates, validation criteria, and environment configuration
- Deployable on standard Vocareum infrastructure with full governance controls
- Tailorable per cohort, partner, event, or individual learner context
- Iterative refinement before publishing
- Compatible with AI Gateway for labs that require governed model access

## **Typical use cases**

Use On-the-Fly Labs when a team needs to stand up a hands-on experience quickly, such as a partner workshop next week, a conference demo, a new cohort starting a topic not yet covered by existing lab inventory, or a customer-specific training scenario. Program leads describe the objective and the platform generates a lab that can be reviewed, refined, and deployed without a multi-week authoring cycle.

## **Public proof / case studies**

Positioned in the April 2026 webinar program and ASU+GSV framing as a core product capability. Early proof is anchored in partner workshop and event delivery use cases where rapid lab creation replaced multi-week authoring timelines.
